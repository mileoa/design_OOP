# Рефлексия
Посмотрел пример использования наследования исходниках django. При длинной цепочке наследования может быть сложно ориентироваться во всех взаимосвязях. Возможно, для упрощения помогут какие-либо утилиты для визуализации. Также композиция в django применяется часто, например queryset. В данном случае quryset является, например, полем generic view. Композиция может использоваться как для хранения данных, так и для реализации сложной логики, которая в теории могла бы быть реализована методами класса, в который она "встроена". Также полиморфизм на примере generic views в django достаточно нагляден. Все view для конкретных действий наследуются от более общих реализаций, которые также можно использовать в качестве шаблонов для собственных view. И все view будут обрабатываться одинаково за исключением прикладной логики, которую можно менять через наследование.
